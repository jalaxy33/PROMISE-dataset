Would you PLEASE consider using the open-source Log4J API instead of
introducing yet another
API in Java. Log4J has existed a long time already, and we have been using
it without any
problems at all in many large production-level application for the last 12
months.

After what I have been able figure out the API of JSR47 is similar to Log4J,
but less powerful and a lot less 'tried and tested'. It also has a (in my
view) bad design flaw, since it can throw exceptions.

Hope you will at least consider using Log4J as the 'standard' in future
JDK's instead.
