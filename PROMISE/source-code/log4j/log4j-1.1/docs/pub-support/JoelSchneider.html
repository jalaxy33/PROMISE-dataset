Subject: Log4j vs. JSR-000047

Dear Sirs:

Please consider adopting Log4j as the standard logging mechanism for Java
1.4, instead of the less mature (and less capable) existing JSR-000047
Logging API Specification.

Log4j is the leading established Java logging mechanism.  It has been
adoped by the Apache Foundation (http://jakarta.apache.org/log4j/) and
enjoys wide acceptance in both open-source and commercial software
packages.  Introduction of a competing logging mechanism would be likely
to cause a split in the Java community, and thereby weaken the Java
platform overall.

Even if it were to cause a delay in the release of Java 1.4, I would urge
you to replace the existing JSR-000047 with Log4j.

Joel
