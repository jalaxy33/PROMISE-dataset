Hello,

I'd like to encourage you to use log4j from the Apache project as the
official logging
API for the JDK 1.4.  When numerous logging packages already exist, it
doesn't
make any sense to create a new API that will only be supported in JDK 1.4
and above.
Log4j works with the JDK 1.1 and above, and has been in use for several
years now,
with massive testing and development work by the open source community.  I
feel that
developers will continue to use log4j for server-side logging because of
superior functionality,
and because it is open source.

Please drop JSR47 from Java 1.4, and announce log4j as the new standard for
Java logging.

Thanks,
Jeff Linwood
