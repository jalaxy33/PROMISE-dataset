Subject: 1.4 Java Logging

After reviewing the jdk 1.4 logging api and discussing it's impact with
others, I'd like to suggest that you consider the Apache Group's log4j
api architecture as a possible alternative to be included in the 1.4
release.

Log4j has been tested rigorously, has been enhanced greatly on a very
tight feedback loop, and is built upon a solid architecture. Log4j also
stresses memory management and performance issues and has a tremendous
array of output destination options.

I will continue to use log4j regardless. I'd just hate to see Sun pass
up the opportunity to utilize a superior piece of open-source software.
