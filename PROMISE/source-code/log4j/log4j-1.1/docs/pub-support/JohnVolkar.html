After a review of the beta JDK 1.4, and the logging API, we have decided
that it is unfortunate that the logging mechanism is not pluggable.  In our
systems we use log4j in a very specific and customized fashion, the JSR47
logging mechanism is wholly unsuited to our needs.  That's fine, but we'd
really like to use the Sun provided api (because of client perceptions, no
*real* reason otherwise) and plug-in our own implementation (which would be
our special configuration of Log4j).

Please, before it's too late, change JSR47 to either:
        A) use Log4j directly
or:     B) clearly define an API via an interface, and make the
implementation pluggable.
