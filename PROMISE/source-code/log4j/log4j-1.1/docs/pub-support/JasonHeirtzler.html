Hello,

I am concerned about the possability that JSR47
may be included in JDK 1.4. If this happens, it
will become the defacto standard and many people
will have no choice but pick JSR47 over log4j,
no matter what the technical merits may be of
one vs the other.

With log4j we have a very workable solution that
the community enjoys, that supports all of us
that are using JDK 1.x, JDK 1.2, and JDK 1.3. log4j
is very well debugged, well optimized, and enjoys
much support from the greater java community.

I would ask you to consider everyone's best
interest and favor this existing package and
not reinvent the wheel.

I sincerely hope we can accomplish this and
build on top of the community's hard work to
make java development the best it can be.

Best regards,
