Subject: Vote for log4j

    I have been developing with Java for over three years now, and am
currently involved in an extensive web project which has relied heavily
on log4j.  In fact, given the intricacies of web program and the
sometimes unexpected behavior inherent in asynchronous programming, I
would not have been able to successfully fix some of the most subtle
bugs had it not been for log4j.  The fact that it is used as a standard
logging package in many Apache projects, upon which I heavily rely,
lends credence to its stability and ease of mind to developers who have
enough to worry about with meeting deadlines.
    I hope you take great care in treating this issue, and make it known
within the Sun community that log4j would be a valuable asset in the
Java arsenal.
