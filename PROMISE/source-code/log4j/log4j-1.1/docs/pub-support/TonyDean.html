Concerning JSR47 public review:

I definitely feel the importance of circumventing the following JSR47 short-coming with Log4j methodology:

*       handler inheritance from ancestor loggers (categories)
*       ability to configure multiple instances of a handler class (restriction of logging to only one file at a time is totally unacceptable)
*       printf-like formatting is a big plus in convenience
*       configuration via XML format

Thank you for your consideration.
