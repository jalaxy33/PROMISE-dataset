Hello,

I would like to add my support to Ceki in his criticism of JSR47:

  http://jakarta.apache.org/log4j/docs/critique.html

CodeMint has used Log4j in many projects and we are quite pleased
with it. In particular we like the focus on performance (if logging
is too expensive it will not be used), ease of configuration at
run time and stability (logging code must not throw exceptions).

While I have not had time to study JSR47 in detail I find many of
Ceki's points disturbing. I hope that you will take the time to
address them before you introduce a standard logging package
as part of the JDK.
