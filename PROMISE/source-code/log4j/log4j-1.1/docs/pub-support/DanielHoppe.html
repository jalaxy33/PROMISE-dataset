Dear Madams and Sirs,

we have been successfully using log4j for more than a year in both
development and production use. We are very satisfied with its comprehensive
featureset and flexible output methods (JMS, SMTP, etc). After having
reviewed JSR47 our development team thinks that it might be worth
considering log4j as a standard logging framework of the Java platform as it
is already very mature and - following feedback on the newsgroups - already
seems to be a de-facto standard for Java server development.
