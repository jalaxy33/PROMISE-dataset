I understand that JDK v1.4 will be shipped with a new
logging api.  I've been using Log4J from the Jakarta
project for quite some time now.  Although these two
api's are very similar, it doesn't make sense to me as
to why sun would "reinvent the wheel".   Log4J has
been around for quite some time, and developers have
been quite satisfied with it.  I would urge your team
to please consider using Log4J as the standard logging
api - I don't see any reason for Sun to compete with
something that is already considered the industry
standard.

Thank you,
Scott Gilpin
