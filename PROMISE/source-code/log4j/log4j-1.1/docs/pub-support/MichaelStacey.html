        After attending JavaOne and reviewing the proposed logging API described in
JSR 47, I wish to register a vote for using the log4j api instead.  Various performance and flexibility features of log4j seem superior to those proposed in
JSR 47.

        In particular, we make heavy use of the ability of log4j to automatically
generate email in particular cases.  With the proposed api in JSR 47, we would need
to rewrite a handler to accomplish the task we are already using in log4j.
